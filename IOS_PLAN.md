# Santro AI — iOS App Technical Plan

Status: PLANNING ONLY — do not build until mobile web + PWA metrics hold and push-alert
use case is validated. (Mobile web + PWA shipped 2026-07; this doc is the v1 plan.)

## Concept
**Santro AI Mobile Terminal** — the AI bubble terminal in your pocket. The app is only
worth building for what the website can't do:

1. **Push alerts**: watchlist price/percent moves (backend `Alert` model already exists),
   AI bubble-risk threshold crossings, hot-ticker entries.
2. **Daily AI bubble brief** as a morning notification (data already generated by the
   takeaways pipeline).
3. Native share sheet for share cards.
4. Later: lock-screen widgets / Live Activities for watchlist + bubble-risk gauge.

## Recommended path
1. **Now**: PWA (shipped). Installable, standalone, offline-honest. No App Store needed.
2. **Next**: **React Native + Expo** when push alerts are validated. Reuses the JS data
   contracts (universe.json / ecosystem.json / crypto.json shapes), fastest to ship,
   EAS handles signing. SwiftUI only if native polish becomes a differentiator.
3. WebView-only wrapper: explicitly rejected (App Store 4.2 risk, no native value).

## API gaps list (build before app code)
1. Token auth (bearer + refresh) alongside cookie sessions.
2. `POST /devices` (register push token), `DELETE /devices/:id`.
3. Server-side alert evaluation worker + APNs sender (respect ~15-min delayed data —
   never claim real-time).
4. Rate limits per token.
5. Versioned API prefix (`/v1/...`) before third-party clients exist.

## App screens (v1)
Onboarding (3 cards: bubble map / hot=attention / delayed-data honesty) → Sign in/up →
Home terminal (bubble map + brief) → Stocks → Crypto → ETFs → Bubble risk → Watchlist
(+alerts) → Valuations → Share cards → Account/preferences.

## Compliance notes
- Quotes delayed ~15 min — repeat the disclaimer in-app; no "live/real-time" language.
- Not financial advice footer on analysis screens.
- App Store finance guidelines: education positioning, no brokerage functions.
